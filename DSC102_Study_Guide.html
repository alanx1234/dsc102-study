This static guide keeps the visualizations, but removes the interactive parts. It is meant to be read, reviewed, printed, and used as a checklist for the repeated DSC 102 problem types.

The exam pattern

  1. Recognize the resource: bits, cycles, cache lines, pages, workers, or bytes.
  2. Write the correct formula or simulate the correct state.
  3. Use units carefully.
  4. Explain the systems consequence in words.

1. How to Study This

To aim for a top score, do not only memorize formulas. You need to know when each formula applies and what the answer means physically.

First pass

Read each section for meaning. Say the idea out loud in plain English.

Example: AMAT is expected memory access time after accounting for hits and misses.

Second pass

Cover each worked example and redo it from scratch.

If you cannot start the problem, write the resource being counted first.

Final pass

Use the checklist at the bottom. Anything that feels slow becomes your review target.

Most lost points come from units, off-by-one simulation mistakes, and mixing hit rates.

4. Digital Representation

Representation questions test whether you understand that the same bits can mean different things depending on the type and interpretation.

Bits and capacity

k bits represent 2k unique patterns minimum bits for N items = ceil(log2 N) 1 byte = 8 bits
Worked example: Minimum bits for 70,000 items.
  1. 216 = 65,536, which is too small.
  2. 217 = 131,072, which is enough.
  3. Answer: 17 bits.

Hex, binary, and addressability

Each hex digit maps to exactly 4 binary bits.

0x7F3A = 0111 1111 0011 1010

A 4-hex-digit address is 16 bits. In byte-addressable memory, that means the address space names 216 bytes.

Maximum addressable memory: 216 bytes = 65,536 bytes = 64 KiB.

Alignment

If instructions are 8 bytes and must be 8-byte aligned, valid addresses are multiples of 8.

PC starts at 0x4000: 0x4000, 0x4008, 0x4010, 0x4018, 0x4020
8-byte alignment fixes the lowest 3 address bits to 000, so only 1/8 of byte addresses are valid instruction addresses.

Two's-complement signed integers

Unsigned k-bit range: 0 to 2k - 1 Signed two's-complement k-bit range: -2k-1 to 2k-1 - 1 8-bit signed range: -128 to 127

If the most significant bit is 1, interpret the value as unsigned minus 2k.

Two's-complement overflow visual: 120 + 10

120 + 10 = 130, but 130 is outside the 8-bit signed maximum of 127.

10000010
10000010 as unsigned = 130 10000010 as signed 8-bit = 130 - 256 = -126
Signed overflow happens when adding two positives produces a negative, or adding two negatives produces a positive.

Custom 6-bit float

bits: 010110 sign = 0 exponent = 102 = 2 fraction = 110 value = (-1)s x 2exp - 1 x (1 + f22-1 + f12-2 + f02-3)
Mantissa = 1 + 1/2 + 1/4 + 0/8 = 1.75. Scale = 22 - 1 = 2. Value = 3.5.

Float concepts

  • Floats are binary scientific notation.
  • IEEE float32 has sign, biased exponent, and mantissa/fraction.
  • Bias lets exponent bits represent negative and positive powers.
  • Floating-point arithmetic is approximate and not always associative.
  • Float16 and quantization trade precision for speed/memory savings.
Bias question: with 4 exponent bits, bias is 24 - 1 - 1 = 7.

Characters, endianness, serialization

ConceptWhat to know
ASCII/UnicodeCharacter encodings map symbols to bit patterns. Unicode supports many writing systems.
Big-endianStores most significant byte first. Example: 0x12345678 stored as 12 34 56 78.
Little-endianStores least significant byte first. Example: 0x12345678 stored as 78 56 34 12.
SerializationConvert an object/data structure into bytes for files or networks; deserialization reverses it.

5. CPU, Memory Hierarchy, and Cache

These questions are mostly weighted averages and expected values. The story is always: CPUs are fast, memory is slower, and misses/stalls dominate.

CPI and runtime

average CPI = sum(fractioni x cyclesi) total cycles = instruction count x average CPI execution time = total cycles / clock rate
Worked example: 2 x 109 instructions at 3 GHz. 40% arithmetic at 1 cycle, 30% memory at 5 cycles, 30% control at 2 cycles.
CPI = 0.4(1) + 0.3(5) + 0.3(2) = 2.5 cycles = 2 x 109 x 2.5 = 5 x 109 time = (5 x 109) / (3 x 109) = 1.67 s

Memory stalls

stall cycles = memory ops x stall cycles per memory op total cycles = base cycles + stall cycles stall percent = stall cycles / total cycles speedup = old cycles / new cycles
1B instructions, 25% memory ops, 40-cycle stall.
base = 1B cycles stall = 0.25B x 40 = 10B cycles total = 11B cycles stall percent = 10/11 = 90.9%

AMAT flow

Average Memory Access Time counts the expected cost of hits and misses through the hierarchy.

L1 accessalways paid
→
L1 miss?only misses go to L2
→
L2 miss?only nested misses go to DRAM
AMAT = TL1 + MRL1(TL2 + MRL2 x TDRAM)
L1 = 2 ns, L1 hit = 0.8, L2 = 12 ns, L2 hit given L1 miss = 0.75, DRAM = 100 ns.
L1 miss = 0.2 L2 miss = 0.25 AMAT = 2 + 0.2(12 + 0.25(100)) AMAT = 2 + 0.2(37) = 9.4 ns

Locality

TypeMeaningExample
TemporalUse same data again soon.Loop variable reused.
SpatialUse nearby data soon.Sequential array scan.

Cache lines and row-major layout

elements per line = cache line bytes / element bytes sequential miss fraction ~= 1 / elements per line
64-byte cache line, float64 values.
64 / 8 = 8 values per cache line miss fraction ~= 1/8 hit fraction ~= 7/8
Row-major loop over columns is cache-friendly. Column-wise access jumps by a large stride and causes many misses.

Why NumPy beats Python loops

Python loop

Interprets bytecode, checks types, handles objects, and repeats overhead per element.

NumPy

Uses homogeneous arrays and compiled kernels over contiguous memory.

Cache

Sequential layout helps the CPU reuse loaded cache lines.

Vectorization

Compiled libraries can use SIMD and optimized BLAS-style routines.

Pipelining and branch misprediction

Pipelining overlaps instruction stages to improve throughput. It does not make the first individual instruction lower latency; the first instruction still must pass through all stages.

perfect k-stage pipeline, stage time = s time for n instructions = (k + n - 1)s
Question-folder example: 5-stage pipeline, 10 ms stage time, first 10 instructions.
time = (5 + 10 - 1) x 10 ms = 140 ms
Branch misprediction idea: if a branch is discovered wrong after stage 3, younger wrong-path instructions already in the pipeline are flushed and restarted. In the review question, the 10-instruction total becomes 160 ms instead of 140 ms.
True/false trap: “Pipelining reduces latency of jobs” is false in the simple pipeline sense. It mainly improves throughput after the pipeline fills.

6. Operating Systems and Scheduling

The OS virtualizes hardware and decides how processes share CPUs, memory, files, and I/O devices.

What an OS does

  • Virtualizes CPU, memory, storage, GPU, and network resources.
  • Manages contention between processes.
  • Provides system calls such as open, read, write, fork, wait, and exec.
  • Protects processes from each other using isolation and privilege levels.

Process states

Blocked becomes ready when the I/O or waited-on event completes.

Processes vs threads

CommunicationIPC/files/sockets/shared memoryShared variables
RiskOne crash is more containedOne bug can corrupt shared state

High-yield OS facts from questions

  • A process moves from blocked to ready when its I/O completes.
  • Two separate processes do not share logical address space by default.
  • CPU scheduling supports multiprogramming and uses CPU time that would otherwise be idle.
  • Low response time is especially important for interactive jobs.
  • Threads share process memory, so shared-state bugs are easier to create.

Scheduling metrics

turnaround = completion - arrival response = first run - arrival waiting = time ready but not running

Average turnaround matters for batch jobs. Response time matters for interactive jobs.

Scheduling algorithms

AlgorithmRuleStrengthWeakness
FCFS/FIFORun in arrival order.Simple.Long jobs can delay short jobs.
SJFRun shortest available job, non-preemptively.Good average turnaround.Needs job length estimate.
SRTF/SRCTFRun shortest remaining job, preemptively.Often best turnaround.Can be unfair to long jobs.
Round RobinEach job gets a time quantum.Good fairness and response.Small quantum means context-switch overhead.

Fork output pattern

For the practice code, remember that code before fork() runs once, while code after fork() runs in both parent and child.

W(A); child = fork(); W(B); if (child) wait(NULL); W(C);
What must happen: A prints once. B prints twice. C prints twice. The parent prints B before waiting, then prints C after the child exits. The child does not wait.
Because scheduling can interleave parent and child differently, valid visible outputs include ABBCC and ABCBC. The exact parent/child B's are indistinguishable on screen.

Round Robin checklist

  1. Use the ready queue in order.
  2. Run the front job for min(quantum, remaining burst).
  3. If the job finishes, record completion time.
  4. If it does not finish, append it to the back of the queue.
  5. Response time is based only on the first time the job ever runs.
If context switch overhead is given, charge it exactly as the problem states. Some questions charge it before every process except the first.

7. Virtual Memory and Paging

Paging questions are about splitting addresses, sizing page tables, and simulating replacement policies.

Address translation

page size = 2k bytes offset bits = k VPN bits = virtual address bits - k virtual address = VPN + offset physical address = PFN x page size + offset
8-bit virtual address, page size 16, map 2 -> 7. Translate VA 35.
VPN = floor(35/16) = 2 offset = 35 mod 16 = 3 PFN = 7 physical address = 7 x 16 + 3 = 115

Page table size

entries = 2VPN bits page table bytes = entries x PTE bytes
32-bit VA, 4 KB pages, 4-byte PTE.
4 KB = 212, so offset = 12 bits VPN bits = 32 - 12 = 20 entries = 220 = 1,048,576 size = 1,048,576 x 4 bytes = 4 MiB

Page tables measured in frames

Some questions ask how many physical frames the OS uses for page tables.

20-bit logical address, 4 KB pages, 16 processes, PTE effectively stored as 4 bytes.
logical pages per process = 220 / 212 = 28 = 256 page table per process = 256 x 4 bytes = 1024 bytes all processes = 16 x 1024 = 16,384 bytes frames used = 16,384 / 4096 = 4 frames

Page replacement policies

PolicyEvictsKey simulation detail
FIFOOldest loaded page.Hits do not change queue order.
LRULeast recently used page.Hits update recency.
OPTPage used farthest in future.Ideal lower bound, not realistic.

Repeated ABCD demand-paging pattern

8. Files, Databases, Storage, and I/O

Storage questions usually count bytes moved or the time paid for seek, rotation, and transfer.

File and filesystem concepts

  • A file is a persistent sequence of bytes plus metadata.
  • A directory maps names to file identifiers/inodes.
  • A filesystem exposes logical file abstractions and maps them to physical disk blocks.
  • Common calls: open, read, write, fsync, close, lseek.

Database vs file

A database ultimately stores bytes in files, but adds a formal data model, schema/metadata, query processing, indexes, reliability, and higher-level operations.

9. Parallelism and Scalable Processing

Parallelism questions test dependency limits, serial bottlenecks, speedup, efficiency, and whether more workers actually help.

Core definitions

TermMeaning
ConcurrencyMultiple tasks in progress over the same time period.
ParallelismMultiple tasks literally executing at the same time.
Task graphTasks connected by dependency edges.
Critical pathLongest dependency chain.
Degree of parallelismLargest number of tasks that can run simultaneously.

Task graph lower bound

Tp >= max(total work / p, critical path) speedup <= T1 / lower bound idle time = p x actual time - total work
Total work = 100, critical path = 40, workers = 4.
work / p = 100 / 4 = 25 lower bound = max(25, 40) = 40 max speedup = 100 / 40 = 2.5x if actual time = 50, idle = 4 x 50 - 100 = 100

Amdahl's Law

Amdahl considers adding more hardware to the same fixed-size workload. It does consider parallelism, but it does not scale the problem size up to match the hardware.

speedup(n) = 1 / ((1 - p) + p/n) max speedup with infinite processors = 1 / (1 - p)
If 75% is parallel and 25% is serial:
8-core speedup = 1 / (0.25 + 0.75/8) = 2.91x infinite-core limit = 1 / 0.25 = 4x

Speedup vs scaleup

IdeaQuestion
SpeedupSame work, more hardware, less time?
ScaleupMore work, proportionally more hardware, similar time?
Amdahl is about speedup for a fixed-size job, not growing the total work to match the hardware.

Dask and dataflow

  • Dask builds task graphs lazily.
  • Execution happens when compute is triggered.
  • Too few partitions means too little parallelism.
  • Too many tiny tasks means scheduler overhead dominates.
  • Use diagnostics to find whether workers are idle, overloaded, or waiting on data.

SIMD, GPU, NUMA, false sharing

ConceptMeaningExam language
SIMDSingle instruction operates on multiple data values.Vectorized NumPy can exploit this under the hood.
GPUMassively parallel hardware for high arithmetic intensity workloads.Bottlenecks include transfer, VRAM, divergence, communication.
NUMARemote RAM can be slower than local RAM.3x remote latency with 100 ns local means 300 ns remote.
False sharingThreads write different values on the same cache line.Cache line bounces between cores, hurting performance.

11. Common Mistakes

  • Mixing bits and bytes.
  • Forgetting byte-addressable means one address names one byte.
  • Using 255 as signed 8-bit max; signed max is 127.
  • Forgetting exponent bias is 2exponent bits - 1 - 1.
  • Declaring overflow without checking representable range/sign rule.
  • Forgetting that L2 hit rate is usually conditional on L1 miss.
  • Saying pipelining reduces single-instruction latency instead of throughput.
  • Assuming fork output has only one possible interleaving.
  • Forgetting to update recency on LRU hits.
  • Computing response time as completion time.
  • Ignoring context switch overhead when given.
  • Counting all row-store bytes as useful when only one column is needed.
  • Ignoring writes in total I/O cost.
  • Assuming parallel speedup is linear.
  • Forgetting the critical path lower bound.
